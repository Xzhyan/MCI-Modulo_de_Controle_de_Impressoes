# MCI - Módulo de Controle de Impressões

> Inicialmente, o projeto visa ser um módulo simples e intuitivo para gerenciar a impressão de imagens de exames médicos (ex.: tomografias, raio-X).
> Com o objetivo de agilizar o processo de impressão, sem causar atraso ou acúmulo.
> Apesar de ser um módulo, sua estrutura já está sendo desenvolvida pensando em, futuramente, tornar o sistema maior para atender a várias necessidades de um hospital ou consultório médico.

## Estrutura do Banco de Dados

### Tabelas
- Cargo (Enfermeiro, Técnico, Médico, Funcionario)
- Funções (Profissional de RaioX, de Tomografia, Recepção, Administração)
- Setores (Salas e setores como ex: Sala de RaioX, sala de CDI)
- Usuários (acesso ao sistema, Admin, Supervisor, Padrão)
- Equipes (grupo de usuarios de uma determiada sala/setor)
- Procedimentos (Tipo do atendimento, exame ou consulta)
- Pacientes (cadastro do paciente)
- Processos (basicamente o atendimento do paciente)
- Documentos (sobre o atendimemnto, sejá docs do paciente ou referente ao seu atendimento, como o de impressão do caso)
